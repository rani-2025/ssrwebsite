üöÄ React Basics & Introduction

What is React?

  • Definition: React is a JavaScript library/framework for building user interfaces
  • Runtime: Runs on Node.js and web servers
  • Architecture: Component-based architecture
  • Purpose: Create interactive user interfaces (UI)
  • Key Concept: Everything user interacts with is called User Interface

React Application Structure

  • Root Component: App.js serves as the main root component
  • Child Components: Other components like MovieGrid.js are child components
  • File Extension: React code uses .jsx files containing both HTML and JavaScript

üìö JavaScript ES6+ Fundamentals

Variable Declarations: var, let, const

var (Legacy - Avoid)

  • Function-scoped (ignores block scope)
  • Hoisted and initialized with undefined
  • Can be redeclared
function example() {
    console.log(x); // undefined (hoisted)
    var x = 5;
    
    if (true) {
        var x = 10; // Same variable, redeclared
    }
    console.log(x); // 10 (function level scope)
}

let (Modern)

  • Block-scoped
  • Hoisted but not initialized (temporal dead zone)
  • Cannot be redeclared in same scope
function example() {
    // console.log(y); // ReferenceError
    let y = 5;
    
    if (true) {
        let y = 10; // Different variable (block scope)
        console.log(y); // 10
    }
    console.log(y); // 5
}

const (Preferred)

  • Block-scoped
  • Must be initialized at declaration
  • Cannot be reassigned
function example() {
    const z = 5;
    // z = 10; // TypeError
    
    const obj = { name: "John" };
    obj.name = "Jane"; // OK - modifying property
    // obj = {}; // TypeError - reassigning
}
Best Practice: Use const by default, let when you need to reassign, avoid var

Function Declarations

Traditional Function Declaration

function addTraditional1(a, b) {
    return a + b;
}
console.log(addTraditional1(2, 3)); // Can call before or after declaration

Function Expression

const addTraditional2 = function(a, b) {
    return a + b;
};
console.log(addTraditional2(1, 2)); // Can only call after declaration

Arrow Functions (Modern)

// Single line
const addArrow3 = (a, b) => a + b;
console.log(addArrow3(4, 5));

// Multi-line
const addArrow4 = (a, b) => {
    return a + b;
};
console.log(addArrow4(6, 7));
Key Difference: Traditional declarations are hoisted, arrow functions are not

Rest Operator (...)

Collects multiple arguments into an array parameter

function restOperator(firstArg, secondArg, ...otherArgs) {
    console.log(firstArg);   // 1
    console.log(secondArg);  // 2
    console.log(otherArgs);  // [3, 4, 5, 6, 7, 8]
}
restOperator(1, 2, 3, 4, 5, 6, 7, 8);

// Array processing example
function addFromArray() {
    const arrValues = [1, 2, 3, 4, 5, 6];
    arrValues.forEach((element, index) => {
        if (index === 0) {
            console.log(element); // First element: 1
        } else {
            console.log(element + arrValues[index - 1]); // Current + previous
        }
    });
}

Spread Operator (...)

Expands arrays/objects into individual elements

const arr1 = [1, 2, 3];
const arr2 = [4, 5, 6];

// Spread arrays
const combined = [...arr1, ...arr2]; // [1, 2, 3, 4, 5, 6]

// Spread in function calls
console.log(...arr1); // 1 2 3 (individual arguments)

// Spread objects
const obj1 = { a: 1, b: 2 };
const obj2 = { ...obj1, c: 3 }; // { a: 1, b: 2, c: 3 }

Rest vs Spread Comparison

Aspect Rest Operator Spread Operator
Purpose Collects multiple elements into array Expands array/object into individual elements
Direction Many ‚Üí One (array) One ‚Üí Many (elements)
Usage Function parameters Function calls, array/object literals
Position Must be last parameter Can be anywhere
Memory Tip: Rest gathers, Spread scatters

Component Fundamentals

  • Definition: Reusable building blocks of React applications
  • Nesting: Components can be nested within each other
  • Modularity: Can be exported and imported between files
  • Naming: Component file names must start with uppercase letter

Component Syntax

// Export default function
export default function Header() {
    return (
        <div>
            <h1>My Header</h1>
        </div>
    );
}

// Import (no braces for default export)
import Header from './components/Header';

  • Single Element: return (<div></div>) ‚úÖ
  • Fragment: return (<><div></div><div></div></>) ‚úÖ
  • Multiple Elements: return(<div></div><div></div>) ‚ùå
Rule: Always return a single element or wrap multiple elements in a Fragment

üîÑ Redux Flow

Redux Architecture Flow

When store.dispatch(action) is called, the control flow follows this sequence:

  1. Reducers: Action goes to all reducers to update the state
  2. Middleware: Middleware (like redux-saga) intercepts the action
  3. Sagas: If there's a matching saga watcher, it triggers the corresponding saga

Example: PRODUCTDETAILS_GET_REQUEST Flow

  1. ProductDetails Reducer: Updates loading state
  2. Redux-Saga Middleware: Intercepts the action
  3. ProductDetails Saga: Handles the API call to fetch product details

Saga Typical Operations

  • Make API call with payload data (e.g., ppid)
  • Dispatch success/failure actions based on response
  • Update store with fetched data
Pattern: This is the standard Redux + Redux-Saga flow for handling async operations like API calls

Basic React Questions

Q1: What is React and why is it used?

Answer: React is a JavaScript library for building user interfaces, particularly web applications. It's used because:

  • Component-based architecture for reusability
  • Virtual DOM for better performance
  • Declarative programming approach
  • Strong ecosystem and community support
  • Backed by Meta (Facebook)

Q2: What is JSX?

Answer: JSX (JavaScript XML) is a syntax extension for JavaScript that allows you to write HTML-like code within JavaScript. Key points:

  • Combines HTML and JavaScript in one file
  • Uses className instead of class
  • Must return a single parent element or Fragment
  • Comments use {/* */} syntax

Q3: What's the difference between var, let, and const?

Answer:

  • var: Function-scoped, hoisted, can be redeclared (avoid in modern code)
  • let: Block-scoped, hoisted but not initialized, cannot be redeclared
  • const: Block-scoped, must be initialized, cannot be reassigned
  • Best Practice: Use const by default, let when reassignment needed

Q4: Explain arrow functions vs regular functions

Answer:

  • Regular functions: Hoisted, can be called before declaration, have their own this
  • Arrow functions: Not hoisted, lexical this, more concise syntax
  • Usage: Arrow functions preferred for callbacks and functional programming

ES6+ Questions

Q5: What are Rest and Spread operators?

Answer:

  • Rest (...): Collects multiple elements into an array (function parameters)
  • Spread (...): Expands array/object into individual elements
  • Memory tip: Rest gathers, Spread scatters
  • Usage: Rest for function parameters, Spread for array/object operations

Q6: How do you create a React component?

Answer:

// Functional Component (Preferred)
function MyComponent() {
    return <div>Hello World</div>;
}

// Arrow Function Component
const MyComponent = () => {
    return <div>Hello World</div>;
};

// Export/Import
export default MyComponent;
import MyComponent from './MyComponent';

React Architecture Questions

Q7: Explain the React project structure

Answer: A typical React project includes:

  • public/: Static files, index.html
  • src/: Source code, components, styles
  • package.json: Dependencies and scripts
  • node_modules/: Installed packages
  • build/: Production build output

Q8: What is the Virtual DOM?

Answer: Virtual DOM is a JavaScript representation of the real DOM that React uses to:

  • Improve performance by minimizing direct DOM manipulation
  • Enable efficient diffing algorithm
  • Batch updates for better performance
  • Provide predictable rendering

Redux Questions

Q9: Explain Redux data flow

Answer: Redux follows unidirectional data flow:

  1. Action: Dispatched from component
  2. Reducer: Processes action and updates state
  3. Store: Holds the updated state
  4. Component: Re-renders with new state
  5. Middleware: Can intercept actions (e.g., Redux-Saga for async operations)

Q10: What is Redux-Saga?

Answer: Redux-Saga is middleware for handling side effects in Redux:

  • Manages async operations like API calls
  • Uses generator functions for better control flow
  • Provides powerful effects like call, put, take
  • Enables complex async flow management

Advanced Questions

Q11: What are React Hooks?

Answer: Hooks are functions that let you use state and lifecycle features in functional components:

  • useState: Manages component state
  • useEffect: Handles side effects and lifecycle
  • useContext: Consumes React context
  • Custom Hooks: Reusable stateful logic

Q12: What's new in React 18?

Answer: React 18 introduces:

  • Concurrent Features: Automatic batching, transitions
  • Suspense: Better data fetching patterns
  • New Hooks: useId, useDeferredValue, useTransition
  • Strict Mode: Enhanced development warnings
  • New Root API: createRoot instead of ReactDOM.render

Q16: What is component lifecycle in React?

Answer: Component lifecycle refers to the phases a component goes through:

  • Mounting: Component is created and inserted into DOM
  • Updating: Component re-renders due to props/state changes
  • Unmounting: Component is removed from DOM
  • In Hooks: useEffect handles all lifecycle phases

Q17: Explain useState Hook with example

import { useState } from 'react';

function Counter() {
    const [count, setCount] = useState(0);

    return (
        <div>
            <p>Count: {count}</p>
            <button onClick={() => setCount(count + 1)}>
                Increment
            </button>
        </div>
    );
}

Q18: What is prop drilling and how to avoid it?

Answer: Prop drilling is passing props through multiple component levels. Solutions:

  • Context API: Share data without passing props
  • Redux: Global state management
  • Component composition: Restructure component hierarchy
  • Custom hooks: Extract and share logic

Q19: Difference between controlled and uncontrolled components

Answer:

  • Controlled: Form data handled by React state
  • Uncontrolled: Form data handled by DOM itself
  • Controlled Example: <input value={state} onChange={handler} />
  • Uncontrolled Example: <input ref={inputRef} />

Q20: What is React.memo() and when to use it?

Answer: React.memo() is a higher-order component for performance optimization:

  • Prevents unnecessary re-renders
  • Only re-renders when props change
  • Use for expensive components
  • Similar to PureComponent for functional components
const ExpensiveComponent = React.memo(({ data }) => {
    return <div>{data}</div>;
});
