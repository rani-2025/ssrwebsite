1) array.find()

Returns the first matching element in the array, or undefined if none match.

import React from "react";

export default function FindExample() {
  const users = [
    { id: 1, name: "Alice" },
    { id: 2, name: "Bob" },
    { id: 3, name: "Charlie" }
  ];

  // Find the user with id = 2
  const foundUser = users.find((user) => user.id === 2);

  return (
    <div>
      <h2>Array.find()</h2>
      <p>Found User: {foundUser ? foundUser.name : "Not Found"}</p>
    </div>
  );
}
  • find() stops at the first match and returns the element itself.
  • Use when you need a single object (e.g., user by id).

2) array.forEach()

Iterates over items for side effects (logging, updating an external variable). It does not return a new array.

  • forEach() is great for actions, not for producing values.
  • Prefer map() when you need a transformed array.

3) array.filter()

Returns a new array containing elements that pass the test.

import React from "react";

export default function FilterExample() {
  const numbers = [1, 2, 3, 4, 5, 6];

  // Keep only even numbers
  const evens = numbers.filter((num) => num % 2 === 0);

  return (
    <div>
      <h2>Array.filter()</h2>
      <p>Even Numbers: {evens.join(", ")}</p>
    </div>
  );
}
  • Non-matching items are excluded; the original array is unchanged.
  • Often paired with map() for rendering filtered lists.

4) array.map()

Creates a new array by transforming each element; length is unchanged.

import React from "react";

export default function MapExample() {
  const numbers = [1, 2, 3, 4];

  // Multiply each number by 2
  const doubled = numbers.map((num) => num * 2);

  return (
    <div>
      <h2>Array.map()</h2>
      <p>Doubled: {doubled.join(", ")}</p>
    </div>
  );
}
  • Primary tool for rendering lists in React.
  • When rendering components in a loop, remember a stable key prop.

  • map() ‚Üí returns a transformed array (same length).

Senior Engineer Interview Q&A

Q1. Difference between map() and forEach() in React rendering?

A: map() returns a new array, so it's perfect for generating React elements in JSX. forEach() doesn‚Äôt return a new array, making it unsuitable for rendering directly. Use map() for rendering lists, and forEach() for side effects.

Q2. Performance considerations when using filter() + map()?

A: Chaining filter().map() is common but creates intermediate arrays. For large datasets, consider a single reduce() pass or memoization with useMemo to avoid unnecessary recalculations on re-renders.

Q3. Why is key important when mapping lists in React?

A: Keys help React identify which elements changed, added, or removed. A stable key improves reconciliation performance and prevents issues like input losing focus. Avoid using array index as key unless the list is static.

Q4. How does immutability affect array methods in React state?

A: Methods like map() and filter() return new arrays, preserving immutability, which is essential for React‚Äôs state change detection. Mutating methods (e.g., push(), splice()) should be avoided on state arrays.

Q5. Can find() and some() be used interchangeably?

A: No. find() returns the actual element, while some() returns a boolean indicating existence. Use some() for checks, find() for retrieval.

Q6. When would you prefer reduce() over map()/filter()?

A: When you want to perform multiple operations in one pass (e.g., filter + transform + aggregate). reduce() provides flexibility but can be less readable, so balance clarity and performance.

Pro tip: Pair filter() then map() to efficiently render only what you need in React lists. For senior-level roles, be ready to discuss immutability, performance trade-offs, and keys in React rendering.
