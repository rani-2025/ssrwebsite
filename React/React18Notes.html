key different between javascript var, let and const

var 1. Function-scoped (ignores block scope) 2. Hoisted and initialized with undefined 3. Can be redeclared function example() { console.log(x); // undefined (hoisted) var x = 5; if (true) { var x = 10; // Same variable, redeclared } console.log(x); // 10 bcs its function level }

let 1. Block-scoped 2. Hoisted but not initialized (temporal dead zone) 3. Cannot be redeclared in same scope function example() { // console.log(y); // ReferenceError let y = 5; if (true) { let y = 10; // Different variable (block scope) console.log(y); // 10 } console.log(y); // 5 }

const 1.Block-scoped 2. Must be initialized at declaration 3. Cannot be reassigned function example() { const z = 5; // z = 10; // TypeError const obj = { name: "John" }; obj.name = "Jane"; // OK - modifying property // obj = {}; // TypeError - reassigning }

Use const by default Use let when you need to reassign Avoid var (legacy, confusing scope rules)

rest operator in es6 javascript

The rest operator (...) in JavaScript collects multiple arguments into an array parameter. function restoperator(firstarg, secondarg, ...otherargs){ console.log(firstarg); // First individual argument.. 1 console.log(secondarg); // Second individual argument.. 2 console.log(otherargs); // Array of remaining arguments into restoperator .. [3,4,5,6,7,8] } restoperator(1, 2, 3, 4, 5, 6, 7,8) Key Points Syntax: ...parameterName (three dots before parameter name) Purpose: Captures all remaining arguments into an array Position: Must be the last parameter Result: otherargs becomes an array containing all extra arguments The rest operator is useful when you don't know how many arguments will be passed to a function, allowing flexible parameter handling.

spread operator in es6

1. The spread operator expands arrays/objects into individual elements. const arr1 = [1, 2, 3]; const arr2 = [4, 5, 6]; // Spread arrays const combined = [...arr1, ...arr2]; // [1, 2, 3, 4, 5, 6] // Spread in function calls console.log(...arr1); // 1 2 3 (individual arguments) // Spread objects const obj1 = { a: 1, b: 2 }; const obj2 = { ...obj1, c: 3 }; // { a: 1, b: 2, c: 3 }

key differences between spread and rest operator

Key Differences: Rest vs Spread Aspect Rest Operator Spread Operator Purpose Collects multiple elements into array Expands array/object into individual elements Direction Many → One (array) One → Many (elements) Usage Function parameters Function calls, array/object literals Position Must be last parameter Can be anywhere // REST: Collecting arguments function restExample(first, ...others) { console.log(others); // Array of remaining args } // SPREAD: Expanding array function spreadExample(a, b, c) { console.log(a, b, c); } const nums = [1, 2, 3]; spreadExample(...nums); // Expands array to individual args Memoryusage : Rest gathers, Spread scatters.

React redux flow

When  store.dispatch(action) is called, the control flow goes through the Redux architecture: 1. Reducers - First, the action goes to all reducers to update the state 2. Middleware - Any middleware (like redux-saga middleware) intercepts the action 3. Sagas - If there's a matching saga watcher, it triggers the corresponding saga For your  PRODUCTDETAILS_GET_REQUEST action, the control would go to: 1. ProductDetails Reducer - Updates loading state 2. Redux-Saga Middleware - Intercepts the action 3. ProductDetails Saga - Handles the API call to fetch product details The saga would typically: * Make an API call with the ppid from the payload * Dispatch success/failure actions based on the response * Update the store with the fetched product data This is the standard Redux + Redux-Saga flow for handling async operations like API calls.
