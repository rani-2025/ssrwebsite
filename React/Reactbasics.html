‚ö° Event Handling in React

Basic Event Handling

In React, event handling uses JSX event attributes similar to HTML, but with camelCase naming:

  • JSX: onClick
  • HTML: onclick
Key Rule: Pass function reference, not function call
  • ‚úÖ Correct: onClick={handleClick}
  • ‚ùå Wrong: onClick={handleClick()}

Event Handler Example

function MyComponent() {
  const handleClick = () => {
    console.log('Button was clicked!');
  };

  return (
    <button onClick={handleClick}>
      Click me
    </button>
  );
}

Template Literals and String Embedding

Use backticks (`) for template literals to embed expressions in strings:

let randomNum = 5;
let userInput = 3;
let message = `Computer number: ${randomNum}, Your guess: ${userInput}`;
console.log(message); // "Computer number: 5, Your guess: 3"

üîÑ State Management in React

useState Hook

useState allows components to have their own state:

import { useState } from 'react';

const LightSwitch = () => {
    const [lightOn, setLightOn] = useState(false);
    
    const handleLight = () => {
        setLightOn(!lightOn);
    };

    return (
        <div>
            <h1>Light is turned on: {lightOn ? 'Yes' : 'No'}</h1>
            <button onClick={handleLight}>Toggle Light</button>
        </div>
    );
};

Aspect Stateful Components Stateless Components

Advanced State Example

const MovieWatchlist = () => {
    const [watchList, setWatchList] = useState([]);
    
    const toggleWatchList = (movieId) => {
        setWatchList(prev => 
            prev.includes(movieId) 
                ? prev.filter(id => id !== movieId)  // Remove if exists
                : [...prev, movieId]                 // Add if doesn't exist
        );
    };
    
    return (
        <div>
            <p>Watchlist: {watchList.length} movies</p>
        </div>
    );
};

Prop Drilling

Prop drilling occurs when you pass data from parent to child to grandchild components. Solutions include:

  • Context API: Share data without passing props
  • Redux: Global state management
  • Component composition: Restructure component hierarchy

‚ùì React Basics Interview Questions & Answers

Fundamental Questions

Q1: What is the difference between props and state?

Answer:

  • Props: Immutable data passed from parent to child components
  • State: Mutable data managed within a component using useState
  • Props flow: Unidirectional (parent to child)
  • State updates: Trigger re-renders

Q2: Why should you pass function reference instead of calling the function in event handlers?

Answer:

  • onClick={handleClick} passes the function reference
  • onClick={handleClick()} executes the function immediately
  • Immediate execution happens on every render, not on click
  • Can cause infinite re-render loops

Q3: What is prop drilling and how can you avoid it?

Answer:

  • Prop drilling: Passing props through multiple component levels
  • Solutions: Context API, Redux, component composition
  • Context API: Provides global state without prop passing
  • Redux: Centralized state management

Q4: Explain the difference between stateful and stateless components

Answer:

  • Stateful: Manage their own state using useState/useReducer
  • Stateless: Receive data only through props
  • Stateful use case: Interactive components (forms, counters)
  • Stateless use case: Display components (headers, cards)

Technical Questions

Q5: What is the Virtual DOM and why is it important?

Answer:

  • JavaScript representation of the real DOM
  • Enables efficient diffing and reconciliation
  • Minimizes expensive DOM operations
  • Improves performance through batched updates

Q6: How does React handle events differently from vanilla JavaScript?

Answer:

  • SyntheticEvents: React wraps native events
  • Cross-browser compatibility: Consistent behavior
  • Event pooling: Reuses event objects for performance
  • CamelCase naming: onClick instead of onclick

Q7: What is the difference between ^ and ~ in package.json versions?

Answer:

  • ^ (Caret): Allows minor and patch updates
  • ~ (Tilde): Allows only patch updates
  • Example: ^1.2.3 allows 1.3.0, ~1.2.3 allows 1.2.4
  • Best practice: Use ^ for most dependencies

Practical Questions

Q8: How do you handle form inputs in React?

function ContactForm() {
    const [formData, setFormData] = useState({
        name: '',
        email: ''
    });
    
    const handleChange = (e) => {
        setFormData({
            ...formData,
            [e.target.name]: e.target.value
        });
    };
    
    const handleSubmit = (e) => {
        e.preventDefault();
        console.log('Form data:', formData);
    };
    
    return (
        <form onSubmit={handleSubmit}>
            <input 
                name="name"
                value={formData.name}
                onChange={handleChange}
                placeholder="Name"
            />
            <input 
                name="email"
                value={formData.email}
                onChange={handleChange}
                placeholder="Email"
            />
            <button type="submit">Submit</button>
        </form>
    );
}

Q9: How do you conditionally render components?

// Method 1: Ternary operator
{isLoggedIn ? <Dashboard /> : <Login />}

// Method 2: Logical AND
{isLoggedIn && <Dashboard />}

// Method 3: If-else with variables
let content;
if (isLoggedIn) {
    content = <Dashboard />;
} else {
    content = <Login />;
}
return <div>{content}</div>;

Q10: What are the rules of hooks?

Answer:

  • Only call at top level: Not inside loops, conditions, or nested functions
  • Only call from React functions: Components or custom hooks
  • Custom hooks: Must start with "use"
  • Consistent order: Same hooks in same order every render
