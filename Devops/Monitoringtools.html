Monitoring and Observalability

1. Monitoring means checking what is wrong. (reactive) 2. Observalability means why the issue happening. (proactive)

pillars of Observalability

1. Metrices - are things which u can measure - let say i want to monitor ec2 instance - what i will do how much memory, storage, network, data, throughput, cpuutilization 2. Logs - every application running produce logs - 3. Traces - how to much it took time start to end will be measured 4- golden signals - using these u can determine ur system is healthy or not 1. Latency - delay between action and response 2. Traffic - request / sec - 3. Error 4. Saturation - the state or process that occurs when no more of something can be absorbed,
