salesforce dev environment setup with vscode

1. salesforce - download https://developer.salesforce.com/tools/salesforcecli 2. then download - vscode 3. vscodev - extensions - salesforce extension pack, salesforce extension pack (expanded) 4. go to terminal - zsh - type - sfdx --version (it should show ) then enter 5. sfdx update (once it completes) 6. now go to terminal enter - sfdx force:org:list > then If you see your org listed under Default Org, Scratch Orgs, or Non-Scratch Orgs, you're connected. you need to authenticate by 7. sfdx force:auth:web:login -a MyDevOrg (MyDevOrg alias to ur org dev, This will open a browser window — log in to your Salesforce account. then enter credentials and authenticate) 8. then run sfdx force:org:list - it should show MyDevOrg in org list 9. test connect by opening -> sfdx force:org:open -u MyDevOrg -> then it will open salesforce developer with org data 10.then pull or deploy metadata - sfdx force:source:pull -u MyDevOrg or sfdx force:source:deploy -p force-app -u MyDevOrg (When to Use source:pull You made changes directly in the Scratch Org (e.g., in Setup UI or Dev Console) You want to pull those changes into your local project You're syncing your local codebase with what exists in the Scratch Org) 11. then test again sfdx force:auth:web:login -a MyDevOrg

Make Sure Extensions Are Installed (VS Code) Install from the Extensions sidebar: ✅ Salesforce Extension Pack ✅ Salesforce Extension Pack(expanded) ✅ Salesforce CLI Integration ✅ Prettier (optional, for formatting)
