Triggers

1. triggers called internal - when some event happens like insert, update, delete 2. called based on the event - adding a record in orders table 3. on triggers there are 2 system tables inserted , deleted. e.g - > orders, customers table -> the momemt order been placed new row will be inserted in orders table, then update custimer table total amount should be updated --create insert trigger-- when order been inserted into orders it will update into customers CREATE TRIGGER update_amount ON orders AFTER INSERT AS BEGIN SET NOCOUNT ON; -- it will not return total records affected messages declare @amt decimal(7,2); -- local variables declare @cnum int; select @cnum=cnum, @amt=amt from inserted; --inserted is a system table in triggers which will have last inserted data update customer set total_orders = total_orders + @amt where customer.cnum = @cnum; END; select * from orders; select * from customer; insert into orders values(3051, 1000, '2025-09-03', 2001);

userdefined Functions

2 type of udf -- all created functions we can see under programability section in SSMS 1. scaler - it will take one or more param but return single value - here we can use function part of select query as a column 2. inline table valued functions (ITVF) - it will return table - multiple values - here we can use function as a table means -- select * from (ITVF function name) 3. in ITVF we can modify output as per requirement where as stored procedure returns fixed output.
