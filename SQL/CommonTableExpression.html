CTE vs View vs ITVF in SQL

1. Common Table Expression (CTE)

Definition: A temporary, named result set used within a single query.

Syntax:

WITH CTE_Name AS (
    SELECT column1, column2
    FROM SomeTable
    WHERE condition
)
SELECT * FROM CTE_Name;
  • Exists only during the execution of one query.
  • Can be recursive.
  • Good for breaking down complex queries.
  • Not reusable or parameterized.

2. View

Definition: A stored SQL query that behaves like a virtual table.

Syntax:

CREATE VIEW View_Name AS
SELECT column1, column2
FROM SomeTable
WHERE condition;
  • Reusable and can be queried like a table.
  • Useful for encapsulating logic and simplifying queries.
  • Cannot accept parameters.
  • Supports security abstraction.

3. Inline Table-Valued Function (ITVF)

Definition: A parameterized function that returns a table using a single SELECT statement.

Syntax:

CREATE FUNCTION dbo.GetOrdersByCustomer (@CustomerId INT)
RETURNS TABLE
AS
RETURN (
    SELECT OrderID, OrderDate
    FROM Orders
    WHERE CustomerID = @CustomerId
);
  • Reusable and parameterized.
  • Can be joined like a table or view.
  • More efficient than multi-statement TVFs.
  • Must be a single SELECT; no procedural logic.

When to Use What?

Use Case CTE View ITVF
One-time logic in a complex query ✅ ❌ ❌
Reusable across queries ❌ ✅ ✅
Accepts parameters ❌ ❌ ✅
Best performance (inlined) ⚠️ Sometimes ⚠️ Sometimes ✅ Often
Recursive querying ✅ ❌ ❌
Encapsulation & abstraction ⚠️ Limited ✅ Strong ✅ Strong
Complex multi-step logic ⚠️ Limited ⚠️ Sometimes ❌ (Use MSTVF instead)

Performance Notes

  • ITVFs are inlined in the query plan — often better performance than multi-statement functions.
  • CTEs might not always be optimized as expected — treat them like subqueries.
  • Views can be indexed if schema-bound, but performance varies.
