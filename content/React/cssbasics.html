<p> vs <span> - Block vs Inline Elements

Key Difference: <p> is a block-level element, <span> is an inline element

Comparison Summary

Feature <p> (Paragraph) <span> (Span)
Display Type Block-level Inline
Line Break Starts on new line Stays on same line
Width Takes full width available Only takes content width
Height/Width Control Can set height and width Cannot set height and width (unless display changed)
Margins Top and bottom margins work Only left and right margins work
Use Case Text paragraphs, content blocks Styling part of text, inline elements

Code Examples

<p> Tag Example (Block-level)

<p>This is a paragraph.</p>
<p>This is another paragraph.</p>
<p>Each paragraph starts on a new line.</p>

Visual Demo:

<span> Tag Example (Inline)

<p>This is <span style="color: red;">red text</span> in a paragraph.</p>
<p>You can <span style="font-weight: bold;">bold</span> or 
<span style="font-style: italic;">italicize</span> parts of text.</p>

Visual Demo:

This is highlighted text in a paragraph.

Multiple spans can be on the same line.

Practical Usage

<!-- Use <p> for paragraphs -->
<p>Lorem ipsum dolor sit amet, consectetur adipiscing elit.</p>

<!-- Use <span> for styling parts of text -->
<p>The price is <span style="color: green; font-weight: bold;">$29.99</span></p>

<!-- Common React/JSX usage -->
<p>Status: <span className="status-active">Active</span></p>

Quick Rules

  • Use <p> for complete paragraphs or text blocks
  • Use <span> to style or target specific parts of text
  • <p> cannot be nested inside another <p>
  • <span> can be nested inside <p>, <div>, or other elements
  • Block elements stack vertically, inline elements flow horizontally

📐 Aspect Ratio

What is it: The proportional relationship between width and height of an element, written as Width:Height (e.g., 1:1, 16:9, 4:3)

Common Aspect Ratios

RatioShapeUsed For
1:1Perfect squareProduct images, profile pics, Instagram posts
16:9Wide rectangleVideos, hero banners, YouTube thumbnails
4:3Slightly wideOld TV format, some tablet layouts
3:2Photo standardDSLR photos, card layouts

Why 1:1 is Popular in E-commerce Grids

  • Grid uniformity — all items align perfectly, text/prices line up beneath each card
  • Responsive friendly — squares scale cleanly on mobile and desktop without awkward cropping

CSS Code

/* Force a square container */
.product-image {
  aspect-ratio: 1 / 1;
  width: 100%;
  object-fit: cover; /* prevents image stretching */
}

/* 16:9 video container */
.video-wrapper {
  aspect-ratio: 16 / 9;
  width: 100%;
}

How to Check Aspect Ratio in DevTools

  1. Right-click on image → Inspect
  2. Hover over the <img> tag in Elements panel
  3. Tooltip shows: dimensions (e.g., 350 × 350 px) and rendered aspect ratio (e.g., 1:1)
  4. Or check Computed tab → look for aspect-ratio CSS property

📏 Intrinsic Size vs Rendered Size

Key: Intrinsic = actual file pixels | Rendered = how big it appears on screen
AspectIntrinsic Size (Natural)Rendered Size (Display)
What is itThe actual pixel dimensions of the image fileThe size the browser displays it at on screen
Who controls itDesigner / image optimization serverCSS rules + user's screen size
How to checkDownload file → check properties, or DevTools tooltipDevTools tooltip or Computed tab
Example350 × 350 px (the file itself)180 × 180 px (shrunk on mobile)

3 Scenarios

Scenario A — Perfect Match (ideal):
  Intrinsic: 350×350 | Rendered: 350×350
  Result: Crisp, no wasted bandwidth

Scenario B — Downscaling (common on mobile):
  Intrinsic: 350×350 | Rendered: 180×180
  Result: Looks fine but downloading more pixels than needed (perf hit)

Scenario C — Upscaling (BAD):
  Intrinsic: 350×350 | Rendered: 600×600
  Result: Browser stretches image → looks BLURRY/pixelated

When Talking to UX Designers

  • Default conversation: Talk in Rendered Size — "The layout needs a 350px square image container"
  • Performance concern: Bring up Intrinsic — "We're serving a 1200px file for a 350px container, wasting bandwidth"
  • Blurry image issue: Bring up Intrinsic — "Container is 500px but source file is only 350px, we need a higher-res asset"

📱 Viewport Sizes & Breakpoints

Viewport: The visible area of a webpage on the user's screen, measured in CSS pixels.

The 3 Main Breakpoint Categories

CategoryWidth RangeDevices
📱 Mobile320px – 480pxiPhones, Galaxy phones, portrait mode
📟 Tablet481px – 1024pxiPads, Surface, landscape phones
🖥️ Desktop1025px and upLaptops, external monitors

Why Viewport Matters

  • Responsive layout: CSS media queries rearrange elements at breakpoints (e.g., product grid goes from 4 columns → 2 → 1)
  • Above the fold: UX designers want key content (price, CTA button) visible without scrolling on initial viewport
  • Fluid design: Modern CSS uses viewport units instead of hardcoded pixels

Viewport Units in CSS

/* vw = viewport width, vh = viewport height */
.hero-banner {
  width: 100vw;    /* full screen width */
  height: 50vh;    /* half the screen height */
}

.sidebar {
  width: 25vw;     /* always 25% of screen width */
}

/* Common responsive breakpoints */
@media (max-width: 480px) {
  /* Mobile styles */
  .product-grid { grid-template-columns: 1fr; }
}

@media (max-width: 1024px) {
  /* Tablet styles */
  .product-grid { grid-template-columns: 1fr 1fr; }
}

@media (min-width: 1025px) {
  /* Desktop styles */
  .product-grid { grid-template-columns: repeat(4, 1fr); }
}

Quick Tip: Test Viewport in DevTools

  1. Open DevTools (F12)
  2. Click the device toggle icon (📱 icon top-left of DevTools)
  3. Select preset devices or manually type viewport width
  4. Watch your layout respond to different sizes
