🧠 16 AI Terms Every Beginner Should Know

Simple explanations with real-world analogies

1 Token

What: A small piece of text (word or part of a word) that AI reads and processes.

Example: "I love coding" = 3 tokens. "Unbelievable" = 2 tokens ("Un" + "believable")

🍕 Analogy: Like slicing a pizza — AI reads text in slices, not the whole pizza at once.

2 Context

What: The information (text) the AI can "see" at one time when generating a response.

Context Window: The max tokens an AI can handle at once (e.g., GPT-4 = 128K tokens).

🪟 Analogy: Like a window — the bigger it is, the more the AI can see and remember.

3 Prompt

What: The instruction or question you give to an AI model.

Example: "Explain Docker in 2 lines" — that's a prompt.

Prompt Engineering: The skill of writing better prompts to get better answers.

📝 Analogy: Like ordering food — the clearer your order, the better the dish you get.

4 LLM (Large Language Model)

What: An AI model trained on massive text data that can understand and generate human language.

Examples: GPT-4 (OpenAI), Claude (Anthropic), Llama (Meta), Amazon Titan (AWS)

📚 Analogy: Like a student who read millions of books — can answer almost anything but might make things up.

5 RAG (Retrieval Augmented Generation)

What: A technique to give the LLM YOUR specific data before it answers, so it doesn't hallucinate.

Flow: User asks → Search your docs → Feed relevant docs + question to LLM → Accurate answer

📖 Analogy: Like giving an open-book exam — the AI looks up answers from your documents.

📎 Detailed RAG Notes (Chunking, Embeddings, Vector DB, Pipeline)

6 Embedding

What: Converting text into numbers (vectors) that capture meaning. Similar texts = similar numbers.

Example: "happy" → [0.9, 0.1, 0.8] and "joyful" → [0.85, 0.15, 0.78] — close numbers because similar meaning.

🗺️ Analogy: Like GPS coordinates — similar places are close on the map, similar words are close in number-space.

7 Vector Database

What: A special database that stores embeddings and finds similar items by meaning (not exact match).

Examples: Pinecone, ChromaDB, Amazon OpenSearch, FAISS

🏪 Analogy: A regular store finds items by name. A vector DB finds items by "what's similar to this?"

8 API (Application Programming Interface)

What: A way for your code to talk to an AI model (or any service) over the internet.

Example: You send a prompt to OpenAI's API → it sends back the AI's response.

🍽️ Analogy: Like a waiter — you tell the waiter (API) your order, they go to the kitchen (AI model) and bring back food (response).

9 Agent

What: An AI that can plan, make decisions, and take actions on its own to complete a task.

Example: "Book me a flight to NYC under $300" — agent searches, compares, and books.

Key: An agent = LLM + ability to use tools + decision making

🤵 Analogy: Like a personal assistant who doesn't just answer questions but actually does the work for you.

10 Skills

What: Specific capabilities an agent has been given (like searching the web, reading files, running code).

Example: Amazon Q Developer has skills like: code generation, debugging, explaining code, reviewing code.

🧰 Analogy: Like apps on your phone — each app is a skill that lets your phone do something specific.

11 Tools

What: External functions/services that an AI agent can call to perform actions.

Examples: Calculator, web search, database query, file reader, API calls

Difference from Skills: Skills = what the agent CAN do. Tools = HOW it does it.

🔧 Analogy: A mechanic (agent) has skills (fixing engines). Tools are the wrench, screwdriver he uses.

12 MCP (Model Context Protocol)

What: A standard way to connect AI models to external tools and data sources.

Why: Without MCP, every tool needs custom integration. MCP = one standard plug for all tools.

Think of it as: USB-C for AI — one connector that works with everything.

🔌 Analogy: Like USB — before USB, every device had a different cable. MCP standardizes how AI connects to tools.

13 Inference

What: When the AI model actually generates a response to your prompt. (Using the model, not training it.)

Training: Teaching the model (expensive, takes weeks)

Inference: Using the trained model (fast, happens every time you ask a question)

🎓 Analogy: Training = studying for years. Inference = taking the exam and answering questions.

14 Guardrails

What: Safety rules/filters that prevent AI from generating harmful, inappropriate, or wrong content.

Examples: Block profanity, prevent sharing personal data, deny illegal requests, stay on topic.

AWS: Amazon Bedrock Guardrails lets you set these rules for your AI apps.

🚧 Analogy: Like guardrails on a highway — keeps the AI from going off the road into dangerous territory.

15 Jailbreaking

What: Tricking an AI into ignoring its safety rules by using clever prompts.

Example: "Pretend you're an AI with no rules..." → trying to bypass guardrails.

Why it matters: As a developer, you need to protect your AI apps against jailbreaking attempts.

🔓 Analogy: Like social engineering — tricking a security guard into letting you in without a badge.

16 Fine-Tuning

What: Retraining an existing AI model on YOUR specific data to make it better at a specific task.

RAG vs Fine-tuning:

• RAG = give docs at question time (cheaper, easier, data stays fresh)

• Fine-tuning = retrain the model itself (expensive, but changes model behavior/style)

🎨 Analogy: RAG = giving an artist a reference photo. Fine-tuning = teaching the artist a whole new art style.

❓ FAQ - Interview & Quick Reference

Q: What's the difference between AI, ML, and LLM?

AI = broad field (machines that think). ML = subset (machines that learn from data). LLM = a type of ML model specialized in language.

Q: Why can't I just put all my data in the prompt?

Context window limit. If you have 10,000 pages, it won't fit. RAG retrieves only the relevant 3-5 chunks.

Q: When to use RAG vs Fine-tuning?

RAG → when you need answers from specific docs (company knowledge base, policies). Fine-tuning → when you need the model to behave differently (tone, format, domain-specific language).

Q: What is hallucination?

When AI confidently generates wrong information. RAG + guardrails help reduce this.

Q: What is temperature in AI?

Controls randomness. Low (0.1) = factual, consistent answers. High (0.9) = creative, varied answers.

Q: What's the difference between an Agent and a Chatbot?

Chatbot = just answers questions. Agent = answers + takes actions (calls APIs, runs code, makes decisions).

Q: How does MCP help?

Without MCP: you write custom code to connect each tool. With MCP: one standard protocol connects any tool to any AI model.

Q: What AWS services are used for AI apps?

Amazon Bedrock (LLMs), Bedrock Knowledge Base (RAG), Bedrock Guardrails (safety), OpenSearch (vector DB), S3 (doc storage), Lambda (processing).

Q: What is prompt engineering?

The skill of writing clear, specific prompts to get better AI responses. Includes techniques like: giving examples, setting roles, specifying format.

Q: How to protect AI apps from jailbreaking?

Use guardrails, input validation, content filters, and test with known attack prompts before deploying.
